# TASK-009 · 上传工作台 UI 改造说明

## 目标

把当前“看起来像拖拽区”的上传页，改成真正的“上传工作台”。

用户核心反馈：

- 现在主视觉像“只能拖拽”
- 点击上传入口不够明确
- 表单、文件选择、状态反馈之间的层级不够清晰

目标体验：

- 用户一进入页面，就知道第一步是“选择图片”
- 文件选中后，立即看到预览和基础信息
- 拖拽上传保留，但降级为辅助方式
- 整个上传流程更像一个发布工作流，而不是一个大号 dropzone

## 不需要 Codex 变更的部分

当前后端已经满足这次 UI 改造，不需要新增接口：

- `POST /api/upload/presign`
- `POST /api/wallpapers`

当前提交链路已支持：

- Presigned URL
- 直传 Cloudflare R2
- 壁纸元数据写入 Supabase
- 授权确认字段
- 变体生成
- AI 标签增强

## 建议的页面结构

### 左侧

- 文件选择主卡
- 大按钮：`选择图片`
- 辅助文案：可拖拽到此处，但不是主操作
- 文件选中后显示：
  - 缩略图
  - 文件名
  - 文件大小
  - 预计尺寸
  - `重新选择` 按钮

### 右侧

- 标题 / 描述 / 标签 / 颜色
- 当前创作者
- 登录邮箱
- 授权确认
- 发布按钮

### 顶部或侧边状态

- `选择文件`
- `上传原图`
- `生成预览`
- `写入作品`

或者至少保留现有状态文案，但以步骤方式更清晰呈现。

## 文案方向

避免再用“把作品拖进来”做主文案。

更推荐：

- `选择一张准备发布的作品`
- `上传图片`
- `开始发布`
- `拖拽可用，但你也可以直接选择文件`

## 现有可复用状态

`components/creator/upload-studio-form.tsx` 里已有：

- `idle`
- `submitting`
- `success`
- `error`

这些状态可以直接用于：

- 按钮禁用
- 步骤高亮
- 成功卡片
- 错误提示

## 视觉方向

- 更像“创作者工作台”
- 更强的结构感
- 明确的主操作按钮
- 文件预览比拖拽提示更重要
- 降低“只是一个虚线框”的感受

## 兼容要求

- 不改接口参数
- 不改上传结果结构
- 不影响登录态校验
- 继续保留拖拽能力
- 移动端也能清晰完成“选择文件 → 填信息 → 上传”

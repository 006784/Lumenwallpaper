# TASK-019 · 首页 Hero 左侧文案区重排说明

## 背景

当前首页 Hero 左侧文案区在深色模式下观感偏弱，用户直接反馈“这片区域不太好看”。

对应区域主要位于：

- `components/sections/hero-section.tsx`

## 当前问题

### 1. 版面重心失衡

- 左侧容器本身很宽，但主要内容只挤在左上角一小列
- 标题、描述、CTA、统计被竖向拉得过开，中间出现大面积“空但不高级”的死区
- 右侧胶片墙已经很强，左侧如果没有足够稳的结构，会显得发虚

### 2. 深色背景层次发闷

- 当前深色背景是暖棕黑渐变，但明暗关系不够明确
- 标题虽然大，但环境底色偏糊，导致整个区域像“蒙了一层旧滤镜”
- 不是高级的暗房感，更像亮度没拉开的脏雾面

### 3. 信息层级没有形成一个主视觉中心

- `每日更新中`、`Issue 01`、主标题、说明文案、CTA、统计卡都在抢注意力
- CTA 与标题之间联系弱，像是被塞在中段的独立模块
- 底部统计卡体积大，但视觉价值不够，反而把 Hero 拉得更散

### 4. 底部数据区太“卡片化”

- 三个统计块现在像普通 dashboard 卡片，不像杂志式首页 Hero 的信息收尾
- 左下角巨大的幽灵 `48K` 装饰字和真实统计数字叠在一起，略显脏乱

## 目标方向

把当前区域从“堆了一组组件”改成真正有重心的编辑部封面。

目标体验：

- 第一眼只看到一个明确的视觉中心
- 标题、说明、CTA 形成连续叙事，不再彼此脱节
- 深色模式更干净、更黑、更有聚光感
- 统计信息降级为“收尾信息条”，不是三张悬浮卡

## 建议改法

### 方案重点 A：重排左侧结构

桌面端建议不要继续用单列 `justify-between` 硬撑满高度。

更推荐：

- 上半区：状态标签 + 主标题 + 简介，做成一个收紧的主视觉组
- 中段：CTA 单独成组，但贴近标题，不要离得太远
- 下半区：统计信息做成横向信息带，压低高度

可以理解成：

`meta`
`title + copy`
`cta`
`stats rail`

而不是现在的：

`meta`
`title`
`cta`
`stats`
`大面积空白`

### 方案重点 B：深色背景改成“更黑更克制”

建议把当前这块背景从“暖棕渐变铺满”改成：

- 近黑底色
- 只在标题附近保留一团很轻的暖金色晕光
- 允许加入极弱的颗粒 / 扫描感 / 竖向胶片纹理，但不能脏

关键是让标题像被一束暖光照亮，而不是整块背景都泛黄。

### 方案重点 C：标题不要再孤零零悬着

当前标题本身不错，但需要更强的容器关系。

建议至少做其中一种：

- 给标题组一个更明确的宽度约束，并和 CTA 绑定成一个垂直内容组
- 或者在标题右侧加极轻的竖向 editorial meta，填掉空区
- 或者把 `Issue 01 / Curated Screens` 拉近到标题组顶部，不要飘在远处

### 方案重点 D：统计区改成编辑式信息条

不建议继续保留三张玻璃卡片。

更推荐：

- 改成一整条横向数据栏
- 或者三个更薄、更像目录索引的小模块
- 数字仍然保留，但标签更轻、边框更弱

同时建议删除或大幅弱化左下角幽灵 `48K`。

## 文案和气质建议

气质关键词：

- 编辑部封面
- 暗房海报
- 克制，不要 dashboard
- 有留白，但不是空
- 高级，不要糊

避免：

- 太多发光
- 太像 SaaS 首页
- 太多盒子
- 太重的毛玻璃感

## 验收标准

- 深色 Hero 左侧在桌面端第一眼有明确主焦点
- 标题、说明、CTA 之间距离明显更紧凑，形成一个整体
- 统计区视觉权重下降，不再抢标题
- 左侧空白不再显得“空洞”
- 与右侧胶片墙并排时，左右气场更平衡

## Codex 备注

本次不需要新增 API、类型或数据层支持。

当前问题是纯 UI / 版式问题，Codex 无需补后端能力。
